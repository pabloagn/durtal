# Ingestion Pipeline

The seed data ingestion pipeline imports the owner's existing library data from Excel and Parquet sources into the Durtal database. It is a Python-based ETL (Extract, Transform, Load) system that runs independently of the Next.js application.

---

2. Locate the actual file or regenerate it.
3. Inspect schema (columns, types, row count).
4. Map fields to Durtal schema following the same pattern as the Excel mapping.
5. Add `scripts/ingest/extract/parquet.py` extractor.

## Pipeline Architecture

```
scripts/
+-- ingest/
    +-- __init__.py
    +-- config.py              Database URL, S3 config, file paths
    +-- models.py              Pydantic models matching Durtal schema
    +-- main.py                CLI entrypoint (click)
    |
    +-- extract/               EXTRACT
    |   +-- __init__.py
    |   +-- excel.py           Read knowledge_base.xlsx (all sheets)
    |   +-- parquet.py         Read consolidated_books.parquet
    |
    +-- transform/             TRANSFORM
    |   +-- __init__.py
    |   +-- authors.py         Name inversion, deduplication, People enrichment
    |   +-- works.py           Work record creation, series assignment, status
    |   +-- editions.py        Edition creation, language mapping, publisher linking
    |   +-- instances.py       Library column mapping to instance records
    |   +-- taxonomy.py        Genre/subject/theme/movement import
    |   +-- occult.py          Occult_Library sheet special handling
    |
    +-- load/                  LOAD
    |   +-- __init__.py
    |   +-- database.py        Connection management, transactions
    |   +-- writers.py         Per-table insert logic with conflict resolution
    |
    +-- enrich/                ENRICH
    |   +-- __init__.py
    |   +-- covers.py          Fetch covers via Google Books / Open Library
    |   +-- metadata.py        Fill gaps (ISBN, page count) from APIs
    |
    +-- reports/               REPORT
        +-- __init__.py
        +-- summary.py         Post-ingestion stats and quality report
```

## Execution Order

Steps must run in dependency order. Each step is idempotent (safe to re-run).

### Step 1: Reference Data

Seed foundational lookup tables with no foreign key dependencies.

| Table | Source Sheet | Rows |
|---|---|---|
| `languages` | Languages | 448 |
| `countries` | Countries_Continents | 262 |
| `centuries` | Validation_Fields | 14 |
| `work_types` | Work_Types | 34 |
| `contribution_types` | Contribution_Types | 50 |
| `sources` | URLs | 14 |

Script: `scripts/ingest/seed_reference.py`

Key logic:
- Skip rows flagged as duplicates (e.g., `ISO_639-1_Is_Duplicate = "Y"`)
- Generate slugs from names (lowercase, hyphenated)
- Compute century start/end years from labels (e.g., "19th Century" -> 1800-1899)
- Upsert with count reporting

### Step 2: Taxonomy

Seed hierarchical and flat classification tables.

| Table | Source Sheet | Rows | Structure |
|---|---|---|---|
| `book_categories` | Book_Categories | 825 | 3-level hierarchy |
| `literary_movements` | Literary_Movements | 217 | 3-level hierarchy |
| `themes` | Themes | 567 | 3-level hierarchy |
| `art_types` | Art_Types | 170 | Flat (filtered to literature-related) |
| `art_movements` | Art_Movements | 86 | Flat |
| `keywords` | Keywords | 287 | Flat |
| `attributes` | Attributes | 70 | Flat (with categories) |
| `subjects` | Subjects | 238 | Flat (with descriptions) |

Script: `scripts/ingest/seed_taxonomy.py`

Key logic:
- Build parent-child trees for 3-level hierarchies
- Insert top-down: roots first, then level 2, then level 3
- Generate full-path slugs for uniqueness (e.g., `fiction/literary-fiction/classics`)
- Handle NULL hierarchy levels (partial definitions)
- Art types filtered by literature-related work types
- Link categories to editions via `edition_genres` using category assignments from `Books.Temporary Tags` or `Books.Book_Type`
- Art types filtered by `Applicable_Work_Type(s)` to include only literature-related entries

### Step 3: Publishing Houses

Seed publisher entities and specialties.

| Table | Source Sheet | Rows |
|---|---|---|
| `publisher_specialties` | Publisher_Specialties | 163 |
| `publishing_houses` | Publishing_Houses | 172 |
| `publishing_house_specialties` | (junction) | derived |

Script: `scripts/ingest/seed_publishers.py`

Key logic:
- Specialties seeded first (no dependencies)
- Country resolution: match `countries.name` to get `country_id`; if no match, store text and leave FK null
- Specialty matching: semicolon-separated values matched exactly to `publisher_specialties.name`

### Step 4: Authors

Import author records from the People sheet.

| Table | Source | Rows |
|---|---|---|
| `authors` | People | 2,541 (filtered) |
| `author_contribution_types` | (junction) | derived |

Script: `scripts/ingest/seed_authors.py`

Key logic:
- **Filter by role**: Include people with ANY literature-related role (Writer, Poet, Editor, Translator, Scholar, etc.). Exclude people ONLY in non-literary roles (Painter, Architect, Composer).
- **Name handling**: Parse `Name` and `Surname` fields into `first_name`, `last_name`. Compute `name` (display) and `sort_name` (inverted for sorting).
- **Type tagging**: Parse semicolon-separated `Type` field, match to `contribution_types`, insert into `author_contribution_types` junction.
- **Date mapping**: `birth_day`, `birth_month`, `birth_year`, Gregorian representation, approximate flags.
- **Deduplication**: Skip flagged rows, check existence by name, upsert with richer data.

### Step 5: Works + Editions

Import the main book inventory.

| Table | Source | Rows |
|---|---|---|
| `works` | Books | 3,905 |
| `editions` | Books | 3,905 |
| `work_authors` | (junction) | derived |
| `edition_genres` | (junction) | derived |

Script: `scripts/ingest/seed_books.py`

**Work decomposition**:
- `title`, `author`, `series`, `series_position`, `description`, `rating`, `notes`
- `catalogue_status`: `'catalogued'` if any Library column populated, else based on Priority field
- `work_type_id`: matched from Work_Types lookup

**Edition decomposition**:
- `title`, `publisher`, `publication_year`, `language`, `page_count`, `edition_name`
- `cover_source_url`, `goodreads_id`

**Author resolution**:
- Parse Author field (may contain multiple authors separated by delimiters)
- Look up in `authors` table by sort name
- Create author record if not found

**Series resolution**:
- Look up by title in `series` table
- Create if not found
- Set `series_id` and `series_position` on work

### Step 6: Occult Library

Import the separate occult/esoteric catalogue.

| Table | Source | Rows |
|---|---|---|
| `works` | Occult_Library | 663 (net new) |
| `editions` | Occult_Library | 663 |

Script: `scripts/ingest/seed_occult.py`

Key logic:
- Mark as `catalogue_status = 'wishlist'` (desired collection, not necessarily owned)
- **Deduplication**: Match on title+author against existing works from Step 5. If match exists, merge metadata (add occult categories) rather than creating a duplicate.
- Handle "Anonymous" as a special author case
- Create "Occult Library" collection and assign entries
- Map 2-level category system to `book_categories` hierarchy or dedicated collection

### Step 7: Additional Editions

Import additional editions for existing works.

| Table | Source | Rows |
|---|---|---|
| `editions` | Book_Editions | 114 |

Script: `scripts/ingest/seed_editions.py`

Key logic:
- Match Title + Author against existing works
- Create edition linked to matched work
- Deduplication by ISBN-13 or (publisher, year) combination

### Step 8: Series

Seed series table and reconcile works.

| Table | Source | Rows |
|---|---|---|
| `series` | Book_Series | 153 |

Script: `scripts/ingest/seed_series.py`

Three phases:
1. Seed `series` table from Book_Series sheet
2. Reconcile: migrate `works.series_name` text values to `works.series_id` FK references
3. Drop deprecated `series_name` column via Drizzle migration

### Step 9: Instances

Create instance records from Library columns.

Script: `scripts/ingest/seed_books.py` (same as Step 5, or separate pass)

| Library Column | Location | Instance Details |
|---|---|---|
| `Library_Physical_MEX` | Mexico City (physical) | Format inferred from `Book_Type` |
| `Library_Physical_EUR` | Amsterdam (physical) | Same |
| `Library_Digital_Main` | Calibre (digital) | Format: `epub` or `pdf` |
| `Library_Mobile_Kindle` | Kindle (digital) | Format: `mobi` or `azw3` |
| `Library_Mobile_iPad` | iPad (digital) | Format: `epub` or `pdf` |
| `Library_Mobile_iPhone` | iPhone (digital) | Format: `epub` or `pdf` |

Logic: For each book row, check each `Library_*` column. If populated (non-null, non-empty), create an instance linking the edition to the corresponding location. A single book with `Library_Physical_MEX=Y` and `Library_Digital_Main=Y` produces two instance rows.

### Step 10: Enrichment (Optional, Rate-Limited)

Post-load metadata gap-filling.

Script: `scripts/ingest/enrich/`

- Batch lookup via Google Books API (1,000/day limit)
- Fallback to Open Library (no rate limit)
- Fetch and process cover images to S3
- Update edition records with fetched data (ISBN, page count, description, cover)

### Step 11: Reporting

Generate ingestion summary.

Script: `scripts/ingest/reports/summary.py`

Output:
- Total works, editions, instances created
- Author deduplication stats
- Records skipped (duplicates, parse errors)
- Coverage: percentage of works with ISBN, cover, description
- Console output + JSON report file

## Field Mappings

### `Books` Sheet -> Durtal Schema

| Excel Column | Durtal Table.Column | Transformation |
|---|---|---|
| `Title` | `works.title`, `editions.title` | Direct (canonical title) |
| `Published_Title` | `editions.title` | If populated and different from `Title`, use this as the edition title (original-language title) |
| `Author` | `authors.name` (via lookup) | Invert "Surname, Name" to "Name Surname" for display |
| `Publisher` | `editions.publisher` | Direct. Cross-reference with `Publishing_Houses` sheet. |
| `Published_Date` | `editions.publication_date` / `publication_year` | Parse to DATE if full date, otherwise extract year. 855/3905 populated. |
| `Published_Language` | `editions.language` | Map to ISO 639-1 using `Languages` sheet. If differs from work's `original_language`, set `is_translated = true`. |
| `Original_Language` | `works.original_language` | Map to ISO 639-1. If not set, infer from author nationality. |
| `Page_Count` | `editions.page_count` | Parse integer |
| `Edition` | `editions.edition_name` | Direct |
| `Description_GenAI` | `works.description` | Direct. 561/3905 populated. |
| `Notes` | `works.notes` | Direct |
| `Priority` | `works.rating` | Map 1-5 directly. 314/3905 populated. |
| `Rating` | `works.rating` | Prefer `Rating` over `Priority` if both exist. 54/3905 populated. |
| `Read` | (not imported) | Reading tracking is a non-goal. |
| `Book_Type` | `works.work_type_id` | Map to `work_types` lookup. Most values are literary types, not bindings. |
| `Series` | `works.series_name` -> `works.series_id` | Lookup/create in `series` table. Cross-reference with `Book_Series` sheet. |
| `Series_Number` | `works.series_position` | Parse to decimal. 18/3905 populated. |
| `Goodreads_Path` | `editions.goodreads_id` | Extract numeric ID from path |
| `Hardcover_ID` | `editions.notes` | Store in notes or custom field if needed |
| `Cover_RAW` | `editions.cover_source_url` | URL to raw cover image. Trigger cover pipeline on import. 111/3905 populated. |
| (derived) | `editions.metadata_source` | Set to `'phantom_canon'` for all imported records |
| `Library_Physical_MEX` | `instances` | Create instance at Mexico City |
| `Library_Physical_EUR` | `instances` | Create instance at Amsterdam |
| `Library_Digital_Main` | `instances` | Create instance at Calibre |
| `Library_Mobile_Kindle` | `instances` | Create instance at Kindle |
| `Library_Mobile_iPad` | `instances` | Create instance at iPad |
| `Library_Mobile_iPhone` | `instances` | Create instance at iPhone |

**Catalogue status derivation**: If any `Library_*` column is populated, set `catalogue_status = 'catalogued'`. Otherwise, if `Priority` is set, mark as `'wishlist'`.

### `People` Sheet -> `authors` Table

| People Column | Authors Column | Notes |
|---|---|---|
| `Complete Name (Surname Name)` | `sort_name` | Canonical match key against `Books.Author` |
| `Complete Name (Name Surname)` | `name` | Display format |
| `Name` | `first_name` | Given name |
| `Surname` | `last_name` | Family name |
| `Type` | `author_contribution_types` (junction) | Semicolon-separated roles |
| `Gender` | `gender` | Direct |
| `Birth_Year` | `birth_year` | Parse integer |
| `Birth_Month` | `birth_month` | Parse integer |
| `Birth_Day` | `birth_day` | Parse integer |
| `Death_Year` | `death_year` | Parse integer |
| `Death_Month` | `death_month` | Parse integer |
| `Death_Day` | `death_day` | Parse integer |
| `Nationality` | `nationality` | Map to ISO 3166-1 alpha-3 |

**Author deduplication**: The `Books.Author` column uses the same format as `People.Complete Name (Surname Name)`. Join on this field to enrich author records. Handle edge cases: multiple authors separated by delimiters, missing entries in People, variant spellings.

### `Occult_Library` Sheet -> Durtal Schema

| Excel Column | Durtal Column | Transformation |
|---|---|---|
| `Title` | `works.title`, `editions.title` | Direct |
| `Author` | `authors.name` | Invert surname-first, same as main Books |
| `Edition/Translation` | `editions.edition_name` | Direct |
| `Edition_Language` | `editions.language` | Map to ISO 639-1 |
| `Original_Language` | `works.original_language` | Map to ISO 639-1 |
| `Type` | Genre or tag | Map to genre or tag |
| `Description` | `works.description` | Direct |
| `Notes` | `works.notes` or `editions.notes` | Direct |
| Category Level 1 | `book_categories` or collection | Create as "Occult Library" collection with sub-collections |
| Category Level 2 | `book_categories` (child) or `subjects` | Link as genre children or subjects |

## CLI Interface

```bash
# Full pipeline (all steps)
uv run python -m scripts.ingest.main --all

# Dry run (validate without writing)
uv run python -m scripts.ingest.main --all --dry-run

# Individual steps
uv run python -m scripts.ingest.main --step taxonomy
uv run python -m scripts.ingest.main --step authors
uv run python -m scripts.ingest.main --step works
uv run python -m scripts.ingest.main --step occult
uv run python -m scripts.ingest.main --step instances
uv run python -m scripts.ingest.main --step enrich --limit 100
uv run python -m scripts.ingest.main --step report

# Specific source file override
uv run python -m scripts.ingest.main --all --excel /path/to/knowledge_base.xlsx

# Reset and reimport (DESTRUCTIVE)
uv run python -m scripts.ingest.main --all --reset
```

Individual seed scripts also support `--dry-run`, `--limit <n>`, and `--table <name>` flags.
